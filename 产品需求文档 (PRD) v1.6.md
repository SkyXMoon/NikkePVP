1. 项目概述
--------------------------------------------------------------------------------
1.1 项目背景
《胜利女神：NIKKE》PVP竞技场高度依赖爆裂技能的启动速度。玩家需要精确计算5名角色组队后，蓄满100%爆裂进度条所需的首轮帧数，并知晓爆裂1/2/3的开启时间。现有社区工具存在精度不足、机制覆盖不全等问题。

1.2 产品目标
构建一个基于帧模拟的高精度充能计算器，完整还原以下机制：
- 不同武器类型的攻击时序（蓄力、飞行时间、暖机等）
- 掩体与爆炸带来的充能倍率
- 穿透、额外伤害等角色特殊属性
- 充能完成后爆裂开启的动画时序
最终让玩家直观获得“充满时间”和“爆裂开启时间”两个核心数据。

1.3 目标用户
- NIKKE PVP 竞技场硬核玩家
- 攻略作者、数据分析者

1.4 术语表
- 帧：最小时间单位，1秒 = 60帧。
- 充能值 (burstGen)：角色单次攻击的基础充能百分比，未乘任何倍率。SG为10颗弹丸合计值。已包含角色自身被动技能，不含队友Buff。
- 首轮爆发：0帧到爆裂条首次 ≥100% 的过程，不计算换弹。
- 站位 (P1~P5)：队伍从左到右的5个固定位置。

3. 数据结构
--------------------------------------------------------------------------------
3.1 角色对象定义 (data.js)

每个角色对象包含以下字段：
{
  id: 1,                     // number，唯一
  name: "红莲",              // string
  weapon: "AR",              // 'SMG'|'AR'|'SG'|'MG'|'SR'|'RL'
  burstGen: 0.45,            // number，基础充能百分比
  chargeSpeedPercent: 0,     // number|null，蓄力速度提升百分比 (0-100)，仅RL/SR
  hasPenetration: false,     // boolean，穿透（本体+掩体2倍充能，RL无效）
  hasExtraDamage: false      // boolean，额外伤害（充能倍率×2）
}

字段说明：
- chargeSpeedPercent：默认0。实际蓄力帧数公式：floor(30 / (1 + chargeSpeedPercent/100)) * 2
- hasPenetration：仅对 SMG/AR/SG/MG/SR 生效，RL 爆炸已包含掩体，穿透忽略。
- hasExtraDamage：全武器生效，最终倍率×2。

4. 核心计算模型
--------------------------------------------------------------------------------
4.1 基本规则
- 0 帧开始，维护每个角色的“下一次充能生效帧”优先级队列。
- 取出最小帧，处理所有该帧事件，累加充能，更新各角色下次充能帧，直至总充能≥100%。
- 安全上限 10000 帧。

4.2 武器攻击时序

4.2.1 即时命中武器（SMG/AR/SG/MG暖机后）
武器    首次充能帧    充能间隔    序列
SMG     0              4           0,4,8,...
AR      0              6           0,6,12,...
SG      0              42          0,42,84,...
MG暖机后 180           2           180,182,184,...
- MG 暖机期（0~179帧）攻击忽略。

4.2.2 SR（立即命中）
- 蓄力时间 = floor(30 / (1 + chargeSpeedPercent/100)) * 2，默认 60 帧。
- 首次充能帧 = 蓄力时间。
- 转身 16 帧，充能间隔 = 蓄力时间 + 16。
- 后续充能帧 = 首次充能帧 + k × 充能间隔。

4.2.3 RL（有飞行时间）
- 蓄力时间同上。
- 首次充能帧 = 蓄力时间 + 飞行时间。
  - P1/P2：飞行 16 帧 → 充能生效帧 = 蓄力时间 + 16
  - P3/P4/P5：飞行 14 帧 → 充能生效帧 = 蓄力时间 + 14
- 充能间隔 = 蓄力时间 + 16。
- 第 k 次充能帧 = 首次充能帧 + (k-1) × 充能间隔。

4.3 充能倍率计算

4.3.1 通用公式
实际充能 = burstGen × 掩体倍率 × 额外伤害倍率
- 掩体倍率：
  - RL：固定 4（P1本体+掩体 + P2本体+掩体）
  - 其他武器：hasPenetration ? 2 : 1
- 额外伤害倍率：hasExtraDamage ? 2 : 1

4.3.2 各武器实际充能总结
- RL：burstGen × 4 × (hasExtraDamage ? 2 : 1) → 无额外 4 倍，有额外 8 倍。
- 其他：burstGen × (hasPenetration?2:1) × (hasExtraDamage?2:1)

4.4 集火目标规则（P2启用，不影响充能）
阵营  武器         第一枪目标  后续目标
进攻  SMG/AR/SG/MG P5          P1
      RL           P1          P1
      SR           P5          P5
防守  SMG/AR/MG    P5          P1
      SG           P5          P5
      RL           P1          P1
      SR           P1          P1
注：此表仅留档，充能计算与目标选择无关。

4.5 爆裂开启时序
当总充能 ≥100% 时的动画等待时间：
- T₀ = 充满帧
- 爆裂1开启 = T₀ + 26
- 爆裂2开启 = T₀ + 58
- 爆裂3开启 = T₀ + 90

5. 界面与交互设计（摘要）
--------------------------------------------------------------------------------
- 左侧 70%：角色卡片网格。
- 右侧 30%：队伍槽位 P1~P5 竖排，下方“清空队伍”按钮。
- 结果区显示：
  - 主显示：充满时间（XX 帧 / Y.YY 秒）
  - 副显示：爆裂1开启（XX 帧 / Y.YY 秒）
  - 爆裂2/3开启帧（小字）
  - 总充能速度（%/s）
- 队伍为空时显示占位提示。
